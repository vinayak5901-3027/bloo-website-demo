# Brand analysis & redesign rationale

This document captures the analysis behind New-Demo-0.1 — the legacy Bloo brand, the strategic
direction in the Brand Book, and how the redesign reconciles them.

## 1. Source material analysed

- **Live site** — `https://www.bloo.io` (current IA, hero "Enterprise Logging Without Compromise",
  ABC-of-cyber-defense pillars, security flow, industries, metrics, light theme).
- **Bloo Brand Book (Feb 2026)** — positioning, palette, typography, voice, five narrative pillars,
  language discipline.
- **Datasheets** — Bloo SIEM (cloud), Bloo On-Prem SIEM, SynthAI, Crucible feature overview.
- **Demo-0.1 baseline** — the prior static build this project supersedes (dark cybersecurity UI,
  same brand tokens).

## 2. Old brand (what we preserved)

| Dimension | Finding |
|---|---|
| Primary color | `#00438D` deep blue — authority, trust |
| Secondary | `#005BC5` — links, interactive |
| Accent | `#FFC500` gold — CTAs and key callouts **only** (kept scarce) |
| Typeface | **Lato** across the board; H1 bold italic |
| Emotional tone | Enterprise, reliable, confident, technical, mature |
| Trust mechanisms | Hard numbers (compression, retention, cost), enterprise proof, SOC 2 |
| Visual direction | Dark cybersecurity interface — premium, "evolution not revolution" |

## 3. Strategic direction (Brand Book) layered in

The Brand Book repositions Bloo from "a SIEM" to **the system of record for Telemetry
Intelligence**. We adopted its language discipline without abandoning the recognizable cybersecurity
product story:

- Lead with **memory, reasoning, control, predictability, agents** (the five locked pillars).
- "SIEM" appears only as a product/use-case (the Bloo SIEM pages), never as the company identity.
- Canonical lines used verbatim ("Telemetry is not exhaust", "Queries answer questions. Bloo
  maintains understanding.").

## 4. Comparison matrix — old vs evolved

| Axis | Legacy framing | New-Demo-0.1 framing |
|---|---|---|
| Category | Cloud SIEM / cybersecurity | System of record for Telemetry Intelligence |
| Hero | "Enterprise logging without compromise" | "Enterprise security, without compromise" + memory thesis |
| Product story | A SIEM product | Four pillars: Datafabric → SIEM → SynthAI → Crucible, one datalake |
| Buyer emotion | Reliability | Reliability **+** inevitability ("telemetry is memory") |
| Conversion | Demo request | Demo request **+** TCO framing, sticky CTA, proof bands |
| Visual identity | Same palette, lighter execution | Same palette, dark premium UI, live-telemetry hero motif |

## 5. Opportunities the redesign captures

1. **One coherent platform narrative** — the four products were scattered; now they form a single
   capability lifecycle (capture → detect → investigate → respond → learn).
2. **Predictable-economics wedge** — "no ingestion penalties" is surfaced as a primary
   differentiator (hero, pricing, SIEM-modernization solution, dedicated blog post).
3. **Information architecture** — clear Platform / Solutions / Industries / Resources / Company
   structure with dropdowns, replacing a flatter menu.
4. **Content depth for SEO + GEO** — product/solution pages with FAQ schema, four long-form posts,
   three case studies, all interlinked.
5. **Conversion surfaces** — sticky mobile CTA, pre-footer CTA band on every page, demo form with
   qualification fields, newsletter capture.

## 6. Why this is "evolution, not revolution"

Same palette, same typeface, same dark cybersecurity feel and the same recognizable proof points —
but a sharper category story, a unified product narrative, deeper content, and a faster, more
accessible, SEO-complete implementation.
